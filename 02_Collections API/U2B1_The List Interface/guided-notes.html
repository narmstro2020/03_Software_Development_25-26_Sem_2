Dynamic Collections for Modern Java Development

What is a List?

A List is an ordered collection that can grow and shrink dynamically. Unlike arrays, you don't need to know the size ahead of time!

üí° Key Insight

Think of a List like a smart array that manages its own size. When you need more space, it automatically grows!

Creating Lists

üìÅ Required Imports
import java.util.List;
import java.util.ArrayList;
Creating an ArrayList
// Recommended: Declare as List, create as ArrayList
List<String> names = new ArrayList<>();

// With different types
List<Integer> scores = new ArrayList<>();
List<Double> prices = new ArrayList<>();
List<Boolean> flags = new ArrayList<>();

‚ö†Ô∏è Wrapper Types Required

Lists can only hold objects, not primitives! Use Integer instead of int, Double instead of double, etc.

Creating Immutable Lists (Java 9+)
// Create with initial values - cannot be modified!
List<String> colors = List.of("red", "green", "blue");

// colors.add("yellow"); // This would throw an exception!

Common List Methods

Adding Elements
List<String> fruits = new ArrayList<>();

fruits.add("Apple");      // [Apple]
fruits.add("Banana");     // [Apple, Banana]
fruits.add(0, "Mango");  // [Mango, Apple, Banana] - insert at index
Accessing Elements
String first = fruits.get(0);     // "Mango"
int count = fruits.size();        // 3
boolean empty = fruits.isEmpty(); // false
Modifying and Removing
fruits.set(1, "Orange");  // [Mango, Orange, Banana] - replace at index
fruits.remove(0);         // [Orange, Banana] - remove by index
fruits.remove("Banana"); // [Orange] - remove by object
fruits.clear();           // [] - remove all
Searching
List<String> items = List.of("A", "B", "C", "B");

boolean hasB = items.contains("B");    // true
int firstB = items.indexOf("B");      // 1
int lastB = items.lastIndexOf("B");  // 3
int notFound = items.indexOf("Z");  // -1 (not found)
Method Description Returns
add(elem) Add to end of list boolean
add(index, elem) Insert at position void
get(index) Get element at index Element
set(index, elem) Replace element Old element
remove(index) Remove by index Removed element
remove(object) Remove first match boolean
size() Get count int
isEmpty() Check if empty boolean
contains(obj) Check if present boolean
indexOf(obj) Find first index int (-1 if not found)
clear() Remove all elements void

Iterating Through Lists

Method 1: Traditional For Loop (when you need the index)
List<String> names = List.of("Alice", "Bob", "Carol");

for (int i = 0; i < names.size(); i++) {
    System.out.println(i + ": " + names.get(i));
}
// Output: 0: Alice, 1: Bob, 2: Carol
Method 2: Enhanced For Loop ‚≠ê (most common)
for (String name : names) {
    System.out.println(name);
}
// Output: Alice, Bob, Carol
Method 3: Iterator (safe for removal)
List<String> mutableList = new ArrayList<>(List.of("A", "", "B", ""));

Iterator<String> iter = mutableList.iterator();
while (iter.hasNext()) {
    String item = iter.next();
    if (item.isEmpty()) {
        iter.remove();  // Safe to remove!
    }
}
// mutableList is now [A, B]
Method 4: forEach with Lambda (Java 8+)
names.forEach(name -> System.out.println(name));

// Or even shorter with method reference:
names.forEach(System.out::println);

‚ö†Ô∏è Don't Modify While Iterating!

Using list.remove() inside a for-each loop causes ConcurrentModificationException! Use an Iterator instead.

ArrayList vs LinkedList

‚úì Use ArrayList (99% of cases)

  • Fast random access O(1)
  • Better memory efficiency
  • Better CPU cache performance

Use LinkedList only if...

  • Frequent add/remove at beginning
  • Need Queue/Deque operations
  • Very large lists with lots of insertions
Both implement List - same interface!
List<String> arrayList = new ArrayList<>();
List<String> linkedList = new LinkedList<>();

// Same methods work on both!
arrayList.add("Hello");
linkedList.add("Hello");

Lists Store References, Not Copies

üí° Key Concept

When you store an object in a List, the List holds a reference (memory address) to that object‚Äînot a copy! This means if you modify the object, the change is visible through the List too.

Modifying Objects Through References
List<Student> roster = new ArrayList<>();
roster.add(new Student("Alice", 85));

// Get a reference to the first student
Student s = roster.get(0);

// Modify through the reference
s.setGrade(95);

// The change is reflected in the list!
System.out.println(roster.get(0).getGrade()); // Prints: 95

‚úì This DOES Modify the List's Object

Student s = roster.get(0);
s.setGrade(95);  // Changes the object!

‚úó This Does NOT Modify the List

Student s = roster.get(0);
s = new Student(...);  // Only changes s!

Reassigning s doesn't affect the list. Use roster.set(0, s) instead.

üß™ Try It: Build a Shopping List

Practice using List methods by completing this shopping list manager:

import java.util.List;
import java.util.ArrayList;

public class ShoppingList {
    public static void main(String[] args) {
        // 1. Create an empty ArrayList of Strings
        List<String> shopping = new ArrayList<>();
        
        // 2. Add these items: "Milk", "Eggs", "Bread"
        shopping.add("Milk");
        shopping.add("Eggs");
        shopping.add("Bread");
        
        // 3. Print the size of the list
        System.out.println("Items: " + shopping.size());
        
        // 4. Insert "Butter" at the beginning (index 0)
        shopping.add(0, "Butter");
        
        // 5. Check if "Eggs" is in the list
        if (shopping.contains("Eggs")) {
            System.out.println("Eggs is on the list!");
        }
        
        // 6. Print all items using enhanced for loop
        System.out.println("\nShopping List:");
        for (String item : shopping) {
            System.out.println("- " + item);
        }
        
        // 7. Remove "Bread" from the list
        shopping.remove("Bread");
        
        // 8. Print final list
        System.out.println("\nFinal list: " + shopping);
    }
}

üìù Key Takeaways

  • ‚úì Lists are dynamic - they grow and shrink automatically
  • ‚úì Declare as List<Type>, create with new ArrayList<>()
  • ‚úì Use wrapper types (Integer, Double) not primitives (int, double)
  • ‚úì add(), get(), size() are your most common methods
  • ‚úì Enhanced for loop is the cleanest way to iterate
  • ‚úì Use ArrayList unless you have a specific reason for LinkedList
